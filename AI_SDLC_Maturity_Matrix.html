📌 How Level Advancement Works

  • Every scoring criterion must meet its threshold — there is no partial averaging.
  • Level 2 requires AI-Assisted Deliverables ≥ 50% as a hard gate, plus all other scoring criteria ≥ L2.
  • Level 3 requires Novel AI Tools in production as a hard gate, plus every criterion at L3.
  • Context questions (Q1, Q2, Q5, Q7) inform the assessment but do not block level advancement.

🔬 Novel vs Commodity AI Tools

  • Commodity: chatbots, code completions, MCP integrations, simple agents, performance analysis tools.
  • Novel: multi-agent orchestration, spec-driven development, on-prem deployment, automated test/defect pipelines, complex agentic workflows.
  • Using Claude Code, Cursor, or Windsurf alone does not qualify as Novel — the workflow matters.
  • Evidence for Novel tools (source control screenshots, architecture docs) is required for L3 assessment.

📊 Evidence Requirements

  • DAU: telemetry screenshot (GitHub Copilot metrics, Azure Monitor) or survey results.
  • Deliverables %: written explanation of how % was derived, Jira AI-tag reports if available.
  • Artifacts: Confluence / SharePoint / GitHub screenshots showing artifact names and dates.
  • Novel tools: source control screenshots showing agentic code, pipeline configs, or architecture.

🤔 Common Questions

  • What counts as a "core role"? Roles that collectively represent > 80% of team headcount — typically Dev, QA, BA/PM.
  • AI Costs restricted by client? Partial credit given — note it in the assessment. Not a blocker for L2.
  • Is Q5 (metrics approach) scored? No — it's context only. Q4 (what you track) is what's scored.
  • Is Q7 (champion text) scored? No — it provides evidence context for Q6 which is scored.
