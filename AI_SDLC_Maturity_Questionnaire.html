What tools/features/ways of working do you use to create deliverables across roles? Core Criterion (Level 3)
Commodity Tools: widely adopted, standardized tools — chatbots, simple agents, code review, MCP integration, performance analysis.

Novel Tools: innovative or experimental — on-prem deployment, complex multi-agent orchestration, spec-driven approaches, automated scenario execution, defect management.
▶ Evidence & Verification Required
How this answer is verified: Evidence of tool usage is primarily important for Novel tools (required for Level 3). For Commodity tools at Level 2, evidence is helpful but not strictly mandatory.
  • Provide screenshots from source control tools or code repositories (GitHub, Bitbucket, etc.) showing Novel tool usage — e.g. multi-agent workflow code, spec-driven scripts, automated pipelines.
  • The artifact names should be clearly visible, logically understandable, and self-explanatory.
  • Add brief explanations in comments describing how the novel approach is being used and which roles it serves.
  • For Level 3 assessment, this evidence is crucial — the assessor will verify that Novel tools are genuinely embedded in core workflows.
